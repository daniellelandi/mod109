📖 Anleitung: Neues HTML-File mit Vim in GitHub hochladen

Diese Anleitung zeigt, wie du eine neue HTML-Datei mit dem Vim-Editor erstellst und in dein GitHub-Repository hochlädst.

🚀 Schritt 1: In das richtige Verzeichnis wechseln

Wechsle in dein Repository-Verzeichnis mod109:

cd ~/mod109

Falls du nicht sicher bist, überprüfe mit:

ls -la

📝 Schritt 2: Neue HTML-Datei mit Vim erstellen

Öffne Vim und erstelle eine neue Datei second.html:

vim second.html

Füge folgenden Code ein:

<!DOCTYPE html>
<html lang="de">
<head>
    <meta charset="UTF-8">
    <title>Zweite Seite</title>
</head>
<body>
    <h1>Willkommen auf meiner zweiten Seite!</h1>
</body>
</html>

Speichern & Beenden:

  1. Drücke ESC, um in den Befehlsmodus zu wechseln.
  2. Gib :wq ein und drücke Enter.

📌 Schritt 3: Datei zu Git hinzufügen

Füge die Datei dem Staging-Bereich hinzu:

git add second.html

Falls du alle Änderungen auf einmal hinzufügen möchtest:

git add .

Überprüfe, ob die Datei zum Commit bereit ist:

git status

📌 Schritt 4: Datei in Git committen

Erstelle einen neuen Commit mit einer Nachricht:

git commit -m "Füge second.html hinzu"

📌 Schritt 5: Datei zu GitHub hochladen

Push die Änderungen in das Remote-Repository:

git push origin main

Falls du eine Fehlermeldung bekommst, überprüfe den Branch:

git branch

Falls nötig, setze den Upstream-Branch:

git push --set-upstream origin main

✅ Schritt 6: Überprüfen, ob die Datei in GitHub ist

Öffne dein Repository im Browser:

https://github.com/<dein-username>/mod109

✅ Falls second.html dort sichtbar ist, war alles erfolgreich!

🎯 Fazit

  • 📂 second.html wurde mit Vim erstellt.
  • 🔄 Datei wurde mit git add & git commit gesichert.
  • 🚀 Datei wurde mit git push zu GitHub hochgeladen.
  • 📌 Datei ist jetzt in deinem Repository auf GitHub sichtbar.

Glückwunsch, du hast erfolgreich eine neue HTML-Datei mit Vim erstellt und in GitHub hochgeladen! 🎉
